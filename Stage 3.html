Lesson 1: The Basics of the Web and HTML

Web Pages

A web page is text document written in a language called HTML, or HyperText Markup Language. People use browser to request data from servers via the Internet. The servers return the information back to the browser which interprets the data.

Coding

Coding happens when programmers write tex in a language that a computer can understand. We learned some simple coding using HTML, Tags, Elements, and Attributes.
Tag : A tag is always contained within angled brackets. Most tags have an opening tag (for example < b > and a closing tag < /b >).Some tags do not require a closing tag (like the < br >, line break tag). They are called "void'' tags.
Element: An element refers to everything within a set of opening and closing tags.
Attribute: An attribute is a property of an element, used to set the href attribute on an anchor tag. Used for making links, URL resources.

Lesson 2: Creating a Structured Document with HTML

Developer Tools

Developer Tools (in the Browser, like Google Chrome) have powerful feauture that will allow you to see the DOM structure of a document and it allows you to dig through the embedded block elements/boxes.

Text Editors (for programming)

Scratchpad, Codepen, and Sublime Text, are editors that programmer's use. They have many powerful features that make programming easier. For example, some text editors will automatically generate a closing HTML tag when you write an opening tag and will let you see in real time the changes you've made.

HTML Elements

HTML elements are either inline or block. Block elements create an "invisible box" around the content inside of them.

DOM

DOM (Document Object Model), the "tree-like structure" of an HTML page. It defines the logical structure of documents and the way a document is accessed and manipulated. In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on...

Lesson 2 - Variables & Strings

A variable in Python is a name that refers to a value stored in a memory location. You can store integers, decimals or characters in these variables.

Python's usage of the "=" sign is not mathematical; it does NOT mean equality.

A Python assignment statement forces the variable on the left hand side to become associated with the value of the expression on the right side. For example: my_variable = 10, where as in a mathematical equation, 5 + 5 = 10 is an assertion that both sides of the equal sign are already, in fact, equal.

The difference between 2 + 2 and "2" + "2" is the first entries are numbers and the ones in quotes are strings. The code 2 + 2 would give you 4, while "2" + "2" would provide and output of "22".

Variables are useful because they minimize the amount of code and lessen repetition.

Lesson 3 - Functions/'Procedures'
A functions takes a value, or values, as its input, processes it and produces an output. def my_fuction(): is an example of a function. It always starts with the word def, followed by any given function name, and then the function parameters in parentheses. These parameters will eventually be replaced by actual values when the function is used or called. To use a function, we write the name of the function followed by the value(s) we want to give it in parentheses. For example, a function named square might take a number as input and produce the square of that number as output. def square (n): (On next line) return n * n. Functions help programmers avoid repetition by minimizing the amount of code. Once defined, it does not need to be defined again and can be reused over and over. A function needs the return keyword to tell Python what the function should produce as output.

Stage 3: Introduction to Abstraction

Stage 3 - Introduction to Abstraction
Python has a module inside its Standard Library called OS (Operating System). Inside are other functions, like listdir and rename. We can read the documentation to see what these functions do, even though we don't really understand how they work in the scenes. This is called Abstraction. Object Oriented Programming (OOP) is a powerful way of writing programs. 'An object in programming is an entity or variable that has two entities attached to it: data and things that act on that data. The data are called attributes of the object, and the functions attached to the object that can act on the data are called methods of the object.'- Johnny Wei-Bing Lin For example: turtle.Turtle() (Turtle graphics is a popular way for introducing programming to kids. It was part of the original Logo programming language developed by Wally Feurzig and Seymour Papert in 1966.) Lower case 'turtle' is a module inside Pythonâ€™s Standard Library. Upper case 'Turtle' is a Class of the object 'turtle'. Class is used like a template or a neatly packaged set of attributes that puts things really well. It allows us to write code very easily and minimizes repetition, which is helpful when programming large projects. (This is called Object Oriented Programming, or OOP). When the module turtle.Turtle() is called (it is almost like calling a function), turtle is defined inside the class Turtle and it creates space in memory for a new instance or a new object of the class Turtle. It can now access all of the rest of the methods predefined like, left, right, forward, backward, circle. These can be used to create graphics. The same module can be called in another instance of turtle with a whole different set of attributes, i.e., it can draw a different shape and it can be a different color etc.. This is known as Instance or Object of the Class. A Class can be compared to a car, i.e. a car which has been predefined to have a body, four wheels, an engine, seats, and steering wheel. An Instance or Object of a car can be a body of a different color, a race car, or one with four doors. It still has the predefined attributes plus its own unique properties.

__init__ is called immediately after an instance of the class is created, known as the Constructor. The first argument of every class method, including __init__, is always a reference to the current instance of the class. By convention, this argument is always named self. Instance Variables are defined inside a method (__init__) and they are local to that instance. Instance Method is a function that is defined inside a class and is associated with an instance.
